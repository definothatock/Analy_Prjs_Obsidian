# Listing Game Components


**"Genre"**
- From "game interaction" framing, these are must:
	1. Player unit dynamics
	2. Mechanics
	
	More Detail (and optionals):
	[[Attribute Reference Table]]


**Player Verbs**
- What are the "Acts" that a player mostly do about.
  Complimentary to Mechanics.
	
	Common example:
	- Jumping
	- Escaping


**Game Verbs**
- what are the common "Acts" that the game/computer would do to the player, if applicable.
  Complimentary to Mechanics.
	
	Common example:
	- Hunt
	- Dropping


**Core Resources**
- The Core resources that would Directly change how the player would play/interact/plan.
	
	Common example:
	- Health Points
	- Main trading currency


## Optional
These are usually hard to know, or less important.

**The Core Rules**
- A unit of logic.
	The underlying "agreements" that shaped the Mechanics and Dynamics.
	
	Common example:
	- Is ...
	- n. should ...
	- If?
	- When?
	etc.
	
	These Rules can be opaqued / hidden from outside view (unlike real games). Might not be as useful as the other sections.


**Game "Characteristic"**
- How does the game "feel" as a whole.

# Independent Discussion

## Game System
***A combination of Mechanics (and Rules), along with highly correlated features*** 
With the above components, what are the main Systems?

-  Common Examples:
	- Movement systems
	- Combat system
	- Inventory System
	- Inventory UI Systems

- Common Questions:
	What do they do (no in-out) ?
	What do they transform (in -> out) ?
	What do they provide (out) ?
	Why Must this system?
